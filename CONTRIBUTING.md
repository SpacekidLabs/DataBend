# Contributing

Thanks for helping improve DATABEND.

## Before You Open a Pull Request

- Keep changes focused and easy to review.
- Update the README when behavior or setup changes.
- Avoid committing generated output such as `dist/`, `node_modules/`, or local virtual environments.

## Suggested Workflow

1. Create a branch for your change.
2. Make the smallest useful edit.
3. Verify the app still runs locally.
4. Open a pull request with a short summary of what changed.

## Notes

This repository contains a few experimental subprojects. If you touch one of those areas, keep the change scoped to that folder and document any new setup steps.
